## Overview

CoreOne CAD is made for FiveM-based dispatch operations that need a web-accessible way to manage incidents and keep response efforts coordinated. It blends CAD-style controls with a modern browser interface, giving operators a practical way to work active calls without depending on a heavy desktop client.

The v2026 release is centered on the core dispatcher routine: seeing incidents as they arrive, following unit assignments, and keeping response decisions anchored to a map-first layout. Since it runs in the browser, it can slot into existing workflows where fast access and low-friction use are important.

## How It Works

CoreOne CAD is meant to sit ahead of an active dispatch process and give operators one central place to handle incidents, manage units, and keep geographic context visible.

Typical workflow:

1. Open the browser app.
2. Review incoming incidents.
3. Assign or update units as events change.
4. Use the map view to keep responses aligned with location.
5. Monitor status changes during the incident lifecycle.

If your deployment includes custom routing or web hosting rules, make sure the browser can reach the main HTML entry point and any supporting assets.
